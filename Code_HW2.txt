For measuring latency, in Part IV A, I used Python's time module, specifically the time.perf_counter() function to get the time before and after I had made the post request to the register endpoint, subtracting the difference to get the total time taken. I then aggregated the times I got into two separate lists (US and EU) of 10 items and took the average for each list to get the average latency. I used this strategy for both the US and EU regions and for both the post request to register and the get request to the list endpoint. 

For the eventual consistency, in Part IV B, in a for loop iterating 100 times, I created a new user object with a unique username based on the current iteration and a createdAt timestamp based on the current time (using datetime). For each user object, I would make a post request to the US server's register endpoint, passing the new user object as the payload. I would then immediately make a get request to the EU server's list endpoint. Then I would get the text response of this get request and see if the user I had just registered was in the list of users. If it didn't exist yet, then I would increment a counter - reporting the result of that counter after all the iterations.
